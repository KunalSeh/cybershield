## 📌 Overview

CyberShield is an intelligent cyberbullying detection system that classifies social media text into **6 categories** using a **multi-model ensemble** backed by **semantic sentence embeddings**. Unlike traditional keyword-based approaches, CyberShield understands the full context and meaning of a sentence — detecting explicit, implicit, and sarcastic cyberbullying.

---

## ✨ Key Features

| Feature | Description |
|---|---|
| 🧠 **Semantic Embeddings** | SBERT `all-mpnet-base-v2` encodes full sentence meaning into 768-dim vectors |
| 🤝 **Multi-Model Ensemble** | XGBoost + SVM + Logistic Regression with majority-vote consensus |
| 📊 **Explainable AI** | SHAP token-level explanations show exactly why the model made a decision |
| ⚡ **Severity Index** | Berkeley-trained Ridge Regressor scores severity 0–10 from human annotations |
| 🕵️ **Implicit Detection** | ML-based sarcasm/implicit language classifier trained on 135k human ratings |
| 📡 **Radar Chart** | Confidence visualized across all 6 classes simultaneously |
| 📈 **Session Dashboard** | Real-time analytics tracking analyses, flags, and severity trends |
| ⚠️ **Borderline Warning** | Low-confidence cases flagged for manual review |

## 🏗️ System Architecture

```
Input Text
    │
    ▼
clean_text()  ←  Minimal preprocessing (URLs, handles, punctuation)
    │              No stopword removal — SBERT needs full context
    ▼
SBERT all-mpnet-base-v2
    │              768-dimensional semantic sentence vector
    ▼
┌───────────────────────────────────────┐
│         Multi-Model Ensemble          │
│   XGBoost │ SVM (C=5) │ Logistic Reg │
│         Majority Vote Consensus       │
└───────────────────────────────────────┘
    │
    ├──→ Category Label (6 classes)
    │
    ├──→ Severity Score (Berkeley Ridge Regressor)
    │
    ├──→ Implicit/Sarcasm Flag (Berkeley LR Classifier)
    │
    ├──→ Radar Chart (all 6 class probabilities)
    │
    └──→ SHAP Explanation (token-level contribution)
```

## 🏷️ Classification Labels

| Label | Description |
|---|---|
| `age` | Cyberbullying targeting someone's age |

## ⚙️ Installation & Setup

### Prerequisites
- Python 3.10 or higher
- 4GB RAM minimum (8GB recommended for training)
- ~2GB disk space for models and datasets

### Step 1 — Clone the repository
```bash
git clone https://github.com/YOUR_USERNAME/cybershield.git
cd cybershield
```

### Step 2 — Create virtual environment
```bash
# Windows
python -m venv cybershield-env
cybershield-env\Scripts\activate

# Mac / Linux
python3 -m venv cybershield-env
source cybershield-env/bin/activate
```

### Step 3 — Install dependencies
```bash
pip install -r requirements.txt
```

### Step 4 — Download datasets
Download the following and place in the project root:

- `cyberbullying_tweets.csv` → from [Kaggle](https://www.kaggle.com/datasets/andrewmvd/cyberbullying-classification)
- `measuring-hate-speech.parquet` → from [HuggingFace](https://huggingface.co/datasets/ucberkeley-dlab/measuring-hate-speech)

`augmented_data.csv` is already included in the repository.

### Step 5 — Train models (run once, ~30-40 mins on CPU)
```bash
python train_model.py
```

This generates all `.pkl` model files automatically.

### Step 6 — Launch the app
```bash
streamlit run app.py
```

Open `http://localhost:8501` in your browser.

## 🔬 Research Contributions

1. **Hybrid Transformer-Classical Pipeline** — SBERT embeddings feed classical ML classifiers, combining contextual understanding with interpretability
2. **Domain-Adaptive Augmentation** — Formal English sentences bridge the gap between Twitter training data and real-world polite-phrasing attacks
3. **Dual-Dataset Severity Scoring** — Severity index trained on 135k Berkeley human annotations, not a hardcoded formula
4. **ML-Based Implicit Detection** — Sarcasm and coded language classifier replacing pure regex heuristics
5. **Multi-Model Disagreement Analysis** — Model consensus percentage surfaces ambiguous borderline cases
